## 3. SLO 실측 설계

측정 창은 지표별 자연 창을 쓴다 (사용자 확정).
기입 형식은 SLO-1 선례 (`56.5%↓ (2026-07-15, 3회 중앙값)` = 값 + 날짜 + 방법) 를 따른다.

| 지표 | 자연 창 | 산출 방법 | 기입 형식 예 |
|---|---|---|---|
| 중복 적재율 | 현재 mart 전체 스냅샷 | dbt `unique` 테스트 (content_hash · url) PASS + SQL 교차 확인 (`GROUP BY … HAVING cnt > 1` 0행) | `0% (2026-07-19, mart N건 기준)` |
| 일일 수집 성공률 | `pipeline_runs` 전 이력 | `success_rate` 평균 (신규 회차 포함 · 회차 수 병기) | `xx.x% (2026-07-19, N회 평균)` |
| 필수 필드 완전성 | 현재 mart 전체 스냅샷 | dbt `not_null` 대상 컬럼의 행 단위 통과율을 SQL 로 직접 산출 (dbt test 는 이진 판정이라 비율 불가) | `xx.x% (2026-07-19, mart N건)` |
| 수집량 이상 감지 | 가동 실적 | 수치가 아닌 상태 서술 — SLO-6 상시 감시 가동 + Discord 실발송 검증 실적 | `가동 (실발송 검증 2026-07-13)` |

규칙 세 가지를 지킨다.

- 실측값이 목표 미달이면 그 값을 그대로 기입하고 각주로 사유를 단다.
- 목표 재조정은 사용자 확인 후에만 한다 (SLO-1 의 ≥ 55% 재조정 선례).
- 측정은 신규 회차 실행 뒤에 한다 (성공률 · 이상 감지에 최신 회차 포함).
