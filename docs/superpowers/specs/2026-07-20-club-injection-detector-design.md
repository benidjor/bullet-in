# 무근거 구단명 검출기 설계 — 번역 게이트 4축 (2026-07-20)

번역 산출물에 원문에 없는 구단명이 주입되는 환각 유형의 검출 · 조치 설계.
실사례는 BBC 라운드업 `9956234a…` — 모델이 로저스의 전 소속 (미들즈브러) 을 학습 지식으로 주입해
원문 (Aston Villa 명시) 과 모순되는 서술을 title_ko · body_ko · summary3_ko 3필드에 만들었고, 독자 신고까지 잠복했다.
진단 · 스윕 실측: `docs/troubleshooting/2026-07-20-translation-club-name-injection.md`.

## 1. 문제

- 기존 게이트 3축이 모두 이 유형을 놓친다.
  - 정방향 환각 검출: 번역 누락 · 왜곡 (귀속 문구 기준) 대상이라 삽입은 범위 밖.
  - 역방향 제목 검출기 (#80): 인명 누락 · 무근거 임대만 커버 — 구단명은 범위 밖.
  - glossary 스윕: 표기 변형 교정 — 미들즈브러는 표기 문제가 아님.
- 세상 지식으로는 "참" 에 가까운 정보라 표면상 그럴듯해 독자 신고 전까지 발견이 어렵다.
- 수동 정정은 upsert 밖의 UPDATE 라 revision 재수집 (content_hash 변경) 시 재번역되며 같은 환각이 재발할 수 있다
— 근본 방어는 enrich 시점 검출기 몫.

## 2. 목표 · 비목표

- 목표: 번역 산출물 4필드 (title_ko · summary_ko · summary3_ko · body_ko) 에 원문 근거 없는 구단명이 등장하면
  검출해 기존 재번역 큐로 보내고, 재발 시 WARNING 잔존 로그를 남긴다 (조치 A안 — 2026-07-20 합의).
- 목표: 소스 언어별 대조 함정을 이중 대조 (한글 표기 or 영문 별칭) 로 해소한다
  (영문 단일 대조는 ko 경로 오탐 — 실측 `cc2c7b58`).
- 비목표: 기존 코퍼스 소급 스윕 (수동 스윕 완료 — 205건 중 실환각 1건 · 정정 완료) · 서빙 보류 메커니즘
  (C안 — 재발 실측이 쌓이면 승격 재검토) · 구단명 표기 교정 (glossary 몫).

## 3. 검출 함수 — `enrich.detect_club_injection`

기존 검출기와 같은 순수 함수를 `enrich.py` 에 추가한다.

```python
def detect_club_injection(parsed: dict, source_text: str,
                          club_map: dict[str, list[str]]) -> list[str]
```

- `parsed`: enrich 결과 dict (title_ko · summary_ko · summary3_ko · body_ko).
- `source_text`: 원문 결합 텍스트 — run.py 기존 `src_text` (title_original + body_source + body_excerpt) 재사용.
- 판정: 사전의 각 한글 구단명에 대해 아래 3조건이 모두 성립하면 의심 목록에 추가.
  1. 4필드 중 하나에 한글 표기 등장.
  2. 원문에 한글 표기 부재 (ko 경로 근거 대조).
  3. fold 된 원문에 영문 별칭이 단어 경계로 전부 부재 (en 경로 근거 대조).
- `_fold_latin` · 단어 경계 매치 (`\b`) 는 기존 검출기 로직 재사용 — Price ⊃ Rice 오분류 방지와 동일 계열.
- 한글 표기 매치 가드: 직전 문자가 한글 음절이면 불인정 ("리즈" ⊄ "시리즈")
— 직후 문자는 조사 결합 ("리즈가" · "리즈의") 이 정상이라 판정에 쓰지 않는다.
  조건 1 (번역 필드 등장) · 조건 2 (원문 한글 근거) 양쪽에 같은 가드를 적용한다.
- 사전 밖 구단은 미검출 (name_map 과 같은 점진 확장 정책).

## 4. 사전 — `config/club_map.yaml` (신규)

glossary (표기 교정) · name_map (인명) 과 목적이 달라 별도 파일로 둔다.

```yaml
clubs:
  미들즈브러: [Middlesbrough, Boro]
  토트넘: [Tottenham, Spurs]
  울버햄튼: [Wolverhampton, Wolves]
```

- 구조: 한글 통용 표기 → 영문 별칭 목록 (정식명 + 통칭).
- 시드: EPL 20개 구단 + 아스날 이적설 빈출 유럽 구단 (레알 마드리드 · 바르셀로나 · 바이에른 · 유벤투스 · 클럽 브뤼헤 · 베식타스 등) 약 30–35개.
- 아스날 미등재 — 전 기사가 아스날 문맥이라 검출 신호가 없다.
- 주석 명기 규칙 2건:
  - 별칭 누락 = 오탐 (원문이 Boro 로만 표기한 기사에서 "미들즈브러" 를 무근거로 오판)
  → 등재 시 통칭 별칭 검수 필수.
  - 짧은 한글 표기는 부분 문자열 오매치 주의 — 선행 결합 (예: "시리즈") 은 §3 가드가 방어하고,
    가드 밖 후행 결합 (인터뷰 ⊃ 인터 · 포르투갈 ⊃ 포르투) 은 셋 중 하나로 대응:
    구별 가능한 긴 표기 등재 (인터 밀란 · AS 로마) / 혼동 상위어의 영문 표기를 별칭에 넣어 접지
    (포르투 ← Portugal — 상위어 문맥이면 통과, 미검출 방향으로만 열려 오탐 안전) / 영문 앵커 없으면 미등재 (릴).
  - 짧은 키는 긴 표기를 부분 문자열로 자동 커버 ("밀란" ⊂ "AC 밀란") — 단독 통용 표기 (밀란 · 인테르) 를 키로 쓴다.

## 5. run.py 와이어링 — 조치 (A안)

`detect_roundup_omission` 과 동일한 패턴으로 기존 enrich 루프에 삽입한다.

- 1차 검출 (retry 아님 — `summary_ko` 미저장 행): `title_ko = None` 저장
→ 기존 재번역 큐 (`rows_missing_translation` 재선별) 재사용, 신규 메커니즘 없음. WARNING 로그.
- 재발 (retry — `summary_ko` 기저장 행): 그대로 저장 + WARNING 잔존 로그 (수동 확인 대상).
- 기존 인명 `suspects` 변수와 분리 유지
— 합치면 body 만 오염된 케이스에 불필요한 원문 제목 폴백이 걸린다.
- 라운드업 (bbc_gossip) 제외 없음 — 실사례 자체가 라운드업이며, 제목 재초점과 달리 무근거 구단명은 라운드업에서도 항상 결함.

## 6. 테스트

기존 검출기 테스트 (tests/test_enrich.py) 스타일로 5종.

1. 주입 검출: 영문 원문에 없는 구단명이 번역 필드에 등장 → 검출.
2. 영문 별칭 근거 통과: 원문이 Boro 로만 표기 → "미들즈브러" 통과.
3. 한국어 원문 근거 통과: ko 원문에 "미들즈브러" 실존 → 통과 (`cc2c7b58` 오탐 재현 회귀).
4. 빈 입력 · 빈 사전 통과.
5. 부분 포함 오매치 방지: 원문 "Leeds" 부재 + 번역에 "시리즈" 만 존재 → "리즈" 미검출 (§3 선행 문자 가드),
   조사 결합 ("리즈가") 은 정상 검출 유지.
6. run.py 와이어링: 기존 게이트 배선 테스트와 같은 수준으로 1차 검출 → title NULL · 재발 → 저장 + 로그 경로 확인.
