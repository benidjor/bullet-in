# SP2-a afcstuff 기자 역추적 · 무료 아웃렛 1순위 승격 설계

afcstuff 2순위 인용 트윗을 인용 기자의 타임라인 역추적으로 무료 아웃렛 원 기사에 연결해 제자리에서 1순위로 승격하는 설계.

## 1. 개요

### 1.1. 배경

SP2-a 착수 맥락과 선행 작업 정리.

- **SP1 (완료 · #24)** — afcstuff 인용 트윗 (`[ @handle ]`)을 2순위 항목으로 수집.
- **SP1.5 (완료 · #27)** — DOM 가상화 저수율 수정 (라이브 인용 1건 → 6건), 역추적 입력 볼륨 확보.
- **게이트 스파이크 (2026-07-03)** — 라이브 역추적으로 3가지 실증.
  - **매칭** — 엔티티 · 시간창으로 원본 트윗 특정, Monga 2회 트윗 오탐을 최근접 시간창으로 해소.
  - **정규식 엔티티 추출** — 선수명 · 구단명 정상 추출, 악센트 (`Guimarães`)만 보정 필요.
  - **무료 아웃렛 카드** — BBC (Mokbel 16/25) · arseblog (gunnerblog 12/25)가 자사 기사 link-card 보유, Athletic (Ornstein)은 페이월.

### 1.2. 목적

이 sub-project가 세우는 것과 다음 단계의 전제 정리.

- **1순위 승격 파이프라인** — afcstuff `[ @handle ]` · 선수명에서 기자 타임라인을 역추적해 무료 원 기사를 fetch · 번역해 1순위로 제공.
- **후속 전제** — 승격 실측 · near-miss 로깅으로 SP2-b (Athletic → fmkorea 능동 연결) 필요성 판단 근거 축적.

### 1.3. 스코프 (슬라이스 A)

이번 sub-project가 다루는 범위 확정.

- **무료 아웃렛만 승격** — 해석된 도메인이 known 무료 아웃렛이면 원문 fetch · 번역으로 1순위 승격.
- **Athletic은 2순위 유지** — 페이월이라 이번 슬라이스에서 승격하지 않고 afcstuff 트윗 그대로 둠.
- **제자리 업그레이드** — 인용 트윗 항목 1개를 성공 시 그 자리에서 1순위로 교체 (별도 항목 방출 안 함).

### 1.4. 스코프 밖

이번에 의도적으로 유보하는 항목 명시.

- **Athletic → fmkorea 능동 연결 (SP2-b)** — SP2가 fmkorea를 검색해 페이월 번역본을 끌어오는 연결.
- **신규 범용 추출 라이브러리** — trafilatura류 도입 (기존 `extract_article_body` 재사용으로 불필요).
- **프로필 bio 파싱** — 기자 프로필의 소속 아웃렛 · 이메일 도메인 추출.
- **직접 소스 tier 통일 리팩터** — 고정 tier 소스를 아웃렛 레지스트리 기준으로 바꾸는 무관 변경.

## 2. 아키텍처

### 2.1. 파이프라인 위치

`XPlaywrightAdapter.fetch()` 한 브라우저 세션 안에서 수집 · 역추적을 마치는 흐름.

```
1. afcstuff 스크레이프 → 인용 트윗 (2순위 RawItem)          [기존 SP1]
2. distinct 기자 (스킵리스트 제외 · 상한 N) 타임라인 스크레이프  [같은 세션 · 쿠키 재사용]
3. 브라우저 닫기
4. 인용 항목별 처리
     ├─ 매칭 성공 + 무료 카드 → t.co 해석 → 본문 fetch → 제자리 승격
     └─ 그 외 → 2순위 유지
5. RawItem 목록 반환
```

- **한 세션 재사용** — afcstuff · 기자 타임라인을 같은 브라우저 · 쿠키로 스크레이프해 오버헤드 절감.
- **본문 fetch는 세션 밖** — 브라우저 닫은 뒤 `httpx`로 기사 URL fetch (X 브라우저와 무관).

### 2.2. 승격 의미론

제자리 업그레이드의 구체 동작 정리.

- **성공 시** — 항목의 `url`을 기사 URL로, `body_source`를 원문으로 교체하고 `outlet` · tier 재산정.
- **실패 시** — 항목을 SP1 그대로 2순위 트윗으로 유지.
- **항목 수 불변** — 인용 트윗당 항목 1개, 승격은 방출이 아니라 교체.
- **번역은 기존 enrich 담당** — SP2는 원문 (en) `body_source`까지만 세움.
  - en → ko 번역 (`body_ko` 생성)은 기존 enrich 단계가 처리 (다른 무료 EN 기사와 동일 경로).

### 2.3. 모듈 구조

브라우저 I/O와 순수 로직을 분리해 테스트성을 확보하는 배치.

- **`x_backtrack.py` (신규)** — 엔티티 추출 · 매칭 · 도메인 라우팅 · tier 규칙을 순수 함수로 모아 단위 테스트 가능하게 둠.
- **`x_playwright.py` (얇게 유지)** — 타임라인 스크레이프 · 승격 조립 오케스트레이션만 담당.
- **기존 재사용** — `_accumulate_tweets` (SP1.5) · `_x_cookies` (SP1) · `extract_article_body` (Tier 2-a).

## 3. 컴포넌트

각 단위의 입력 · 출력 · 의존을 독립적으로 정의.

### 3.1. 타임라인 스크레이퍼

기자 핸들에서 최근 트윗을 카드 정보까지 뽑는 단위.

- **입력 · 출력** — `handle` → 트윗 목록 (`text` · `created_at` · `status_id` · `card_href` · `card_domain`).
- **구현** — SP1 스크롤 루프 · `_accumulate_tweets` 재사용 + link-card 추출 JS 추가.
- **의존** — Playwright 페이지 · X 쿠키.

### 3.2. 엔티티 추출

트윗 텍스트에서 인명 · 구단명 후보를 뽑는 순수 단위.

- **입력 · 출력** — 텍스트 → 이름 후보 목록.
- **규칙** — 연속 대문자 시작 단어 (2어 이상)를 후보로, 유니코드 문자 포함해 악센트 보존.
- **합격 기준** — `Bruno Guimarães`가 `Bruno Guimar`로 잘리지 않음.

### 3.3. 매처

기자 타임라인에서 afcstuff가 인용한 원본 트윗 1건을 특정하는 순수 단위.

4단계 규칙으로 후보를 좁히고 정책 Y로 오탐을 배제.

```
① 후보 거르기 : 기자 트윗 중 afcstuff보다 먼저 + 시간창 W 이내만 남김
② 랭킹        : afcstuff 텍스트와 토큰 겹침 순으로 정렬
③ 선택        : 겹침이 임계 T 넘는 최상위 1건, 아무도 못 넘으면 매칭 실패
④ 승격 판정   : 그 트윗의 카드가 무료 아웃렛이면 승격, 아니면 2순위
```

- **정책 Y (정밀 우선)** — 매칭된 바로 그 트윗의 카드만 사용.
  - 최상위 매칭 트윗에 카드가 없으면 근처 트윗에 카드가 있어도 승격하지 않음.
  - 근거는 같은 선수의 다른 하위 스토리 카드를 잘못 붙이는 오탐 배제 (afcstuff-aggregator-direction의 fuzzy 검색 기각과 동일 정신).
- **입력 · 출력** — afcstuff 트윗 · 기자 트윗 목록 → 매칭 트윗 또는 `None`.

### 3.4. t.co 해석 · 도메인 라우터

단축 링크를 실 URL로 풀어 아웃렛 · fetchability를 판정하는 단위.

- **t.co 해석** — `httpx` 리다이렉트 팔로우로 실 URL · 도메인 획득.
- **도메인 라우터** — 도메인 → 아웃렛명 또는 페이월 또는 미등록.
  - `bbc.co.uk` → BBC (무료 · 승격).
  - `nytimes.com/athletic` → Athletic (페이월 · 2순위).
  - 미등록 도메인 (아웃렛 맵에 없음) → 2순위 + 도메인 로깅.

### 3.5. 본문 fetcher

무료 기사 URL에서 본문을 추출하는 단위 (fmkorea 무료 경로와 동일 방식).

- **입력 · 출력** — 기사 URL → 본문 텍스트.
- **구현** — 기존 범용 `extract_article_body` (`meta.py`) 재사용, 도메인별 selector 불필요.
- **아웃렛 필수** — 도메인이 아웃렛 맵에 있어야 승격 (fmkorea가 아웃렛 미확정 시 skip하는 것과 동일).
- **미등록 · 빈 본문** — 맵에 없거나 본문이 비면 승격하지 않고 2순위 + 로깅.

### 3.6. tier 해석

승격 항목의 공신력 등급을 내는 단위 (fmkorea 규칙 재사용).

- **규칙** — 기자 먼저 · 아웃렛 폴백 · 최하 4.
  - 인용 `@handle`이 등록 기자면 그 기자 tier.
  - 아니면 해석 도메인이 등록 아웃렛이면 그 아웃렛 tier.
  - 둘 다 없으면 4.
- **일관성** — fmkorea 어댑터의 tier 산출과 동형이라 발견 소스 간 등급 기준 통일.

### 3.7. 승격 조립

매칭 · 본문 · tier를 합쳐 RawItem을 제자리 승격하는 단위.

- **입력 · 출력** — 원 인용 항목 · 매칭 결과 · 본문 → 승격된 RawItem.
- **필드 교체** — `url` · `body_source` · `outlet` · tier 갱신, 실패면 원 항목 반환.

## 4. 데이터 흐름

인용 1건이 승격 또는 2순위로 갈리는 전 과정.

```
afcstuff 인용 트윗  [ @handle ] + 선수명
        │
        ▼  기자 타임라인 스크레이프 (핸들별 1회 · 상한 N)
   기자 트윗 목록 (카드 포함)
        │
        ▼  매처 (①~④)
   ┌────────────┴────────────┐
   매칭 + 무료 카드            매칭 실패 · 카드 없음 · Athletic · 미등록
        │                          │
        ▼  t.co 해석 · 본문 fetch    ▼
   1순위 승격 (url=기사 · body      2순위 유지 (afcstuff 트윗)
   · outlet · tier)                + near-miss 로깅
        │
        ▼  mart 적재 (URL UNIQUE 중복 제거)
   직접 BBC 수집과 같은 URL이면 자동 병합
```

## 5. 라우팅 · 폴백

조건별 처리 · 로깅 여부 정리.

| 조건 | 처리 | 로깅 |
| - | - | - |
| 매칭 트윗 카드 = 무료 known 아웃렛 | 1순위 승격 (fetch · 번역) | - |
| 도메인 = `nytimes.com/athletic` (페이월) | 2순위 유지 (SP2-b 유보) | 도메인 |
| 매칭 트윗 카드 없음 (정책 Y) | 2순위 유지 | near-miss |
| 매칭 실패 (겹침 < T) | 2순위 유지 | near-miss |
| 도메인 미등록 (아웃렛 맵에 없음) | 2순위 유지 | 도메인 |
| 타임라인 스크레이프 실패 | 해당 항목만 2순위 강등 | 오류 |

## 6. 설정 산출물

### 6.1. 도메인 · 아웃렛 맵

승격 대상 무료 아웃렛을 도메인으로 식별하는 매핑 (selector 없음).

- **형태** — 도메인 → 아웃렛명 (본문은 범용 추출기라 selector 불필요).
- **사전 시딩** — `credibility.yaml`의 무료 아웃렛 약 17개에 도메인 부여.
  - 대상 = BBC · Evening Standard · The Sun · Daily Mail · The Telegraph · The Times · The Guardian · Sky Sports · football.london · 90min · HITC · ESPN · Goal.com 등.
  - Athletic은 페이월이라 제외.
  - 아웃렛명으로 `credibility.yaml` outlet tier 조회 · 서빙 표시.
- **성장** — 미등록 도메인 로깅을 근거로 맵 점증 추가.

### 6.2. 스킵리스트 · 파라미터

X 읽기를 아끼고 튜닝 여지를 남기는 값.

- **트윗-only 스킵리스트** — `@LatteFirm` 등 기사가 없는 팟캐스트 핸들 역추적 제외.
- **파라미터 기본값 · 라이브 튜닝** — 시간창 `W` · 겹침 임계 `T` · 타임라인 깊이 · 기자 스크레이프 상한 `N`.
  - 스파이크 표본이 작아 지금 확정하지 않고 기본값 + near-miss 로깅으로 관측 · 조정.
  - 상한 초과분은 무음 절단 없이 드롭 로깅.

## 7. 오류 처리 · 격리

역추적 실패가 수집 전체를 죽이지 않게 하는 설계.

- **항목 단위 강등** — 기자 스크레이프 실패 (쿠키 만료 · 타임아웃 · 비공개 · 정지 · 레이트리밋)는 해당 인용만 2순위로 강등.
- **소스 격리 준수** — 예외를 위로 전파하지 않아 afcstuff fetch · 다른 소스 · 파이프라인이 계속 진행 (`gather_all` 격리).
- **본문 fetch 실패** — 승격 포기하고 2순위 유지 + 로깅.

## 8. 중복 제거 · tier 상호작용

발견 소스 간 병합 · 등급 일관성 정리.

- **무료 URL 자동 병합** — 승격 기사 URL이 직접 BBC 수집과 URL UNIQUE로 한 행에 병합.
- **tier 일관성** — SP2가 fmkorea와 같은 tier 규칙 (기자 먼저 · 아웃렛 폴백)이라 발견 소스 간 등급 기준 통일.
- **직접 소스 고정 tier 차이** — 고정 tier 소스와의 값 차이는 fmkorea에도 있는 수용된 특성, 이번 리팩터 대상 아님.

## 9. 성공 기준

3층으로 검증 가능한 목표 정의.

### 9.1. 단위 테스트 (TDD)

네트워크 없이 순수 로직을 각각 검증.

- **엔티티 추출** — 악센트 이름 미절단.
- **매처** — 최고 겹침 트윗 선택 · 임계 미만 시 `None` · 정책 Y (카드 없으면 승격 안 함) · 후행 트윗 제외.
- **도메인 라우터** — 무료 · 페이월 · 미등록 분기.
- **tier 규칙** — 기자 먼저 · 아웃렛 폴백 · 4.
- **t.co 해석** — 리다이렉트 모킹으로 실 URL 확인.

### 9.2. 라이브 종단 검증

모킹이 못 잡는 실 DOM · 카드 · 본문 추출을 실행으로 확인.

- **최소 1건 승격** — 실 인용이 승격돼 `url`=기사 · `body_ko` 채움 · `outlet` · tier 정확.
- **실패건 2순위 유지** — Athletic · 카드 없음 · 매칭 실패가 잘못 승격되지 않음.
- **로깅** — near-miss · 미등록 도메인 · 드롭 기록.

### 9.3. 무회귀 · 격리

기존 동작 보존과 실패 시 안전 강등 확인.

- **기존 스위트 그린** — SP1 · 파이프라인 무회귀.
- **안전 강등** — 스크레이프 실패 시뮬레이션에서 항목만 2순위로 떨어지고 예외 미전파.

### 9.4. 승격 비율 비게이트

비율을 합격선으로 걸지 않는 이유 명시.

- **가변 요인** — 승격 비율은 라이브 이적 볼륨 · 아웃렛 믹스 (월드컵 영향)에 의존.
- **관측 용도** — 로깅으로 비율을 추적해 튜닝 · SP2-b 필요성 판단의 입력으로만 사용, 합격 판정엔 미사용.
