# 요약 말투 정리 — '합니다체' 잔존 해소 설계 (2026-07-15)

v1 마감 3트랙 중 ① (로드맵 `../2026-06-28-v1-completion-roadmap.md` "v1 마감 범위").
요약 필드의 존댓말 ('합니다체') 잔존을 신규 생성분과 기존 저장분 양쪽에서 0으로 수렴시킨다.

## 배경 · 문제

- 프롬프트 (`enrich.py`의 `SUMMARY_PROMPT` · `TRANSLATE_PROMPT` · `PARAPHRASE_PROMPT`)에는 평어체 (종결어미 '~다') 지시가 이미 있으나, 모델이 간헐적으로 무시한다.
- 결과: `summary_ko` · `summary3_ko` 일부에 '~합니다 · ~습니다' 체가 남아 인덱스 · 상세 페이지에 노출된다.
- v1 마감 잣대 (공개 데모 완성도)에서 라이브 화면에 바로 보이는 텍스트 품질 항목이라 v1에 포함됐다.

## 범위

- 대상 필드: `summary_ko` · `summary3_ko`만.
- 제외: `body_ko` — 본문 번역에는 선수 인용문 ("우리는 준비돼 있습니다")의 존댓말이 정상 포함되므로 검출 대상이 아니다.
- 제외: `title_ko` — 명사형 제목체라 해당 없음.

## 설계

### 1. 검출기 (순수 함수)

- 요약 텍스트에서 합니다체 · 해요체 종결어미 (~습니다 · ~합니다 · ~해요 · ~예요 등)를 검출한다.
- **인용부호 안 텍스트는 검출에서 제외한다** — 요약에 짧은 인용이 들어올 수 있고, 이 예외가 없으면 정상 요약을 무한 재생성하는 함정이 된다.
- 단위 테스트로 케이스를 고정한다: 합니다체 검출 · 인용문 통과 · 평어체 통과.

### 2. 프롬프트 강화

- 평어체 지시에 금지/올바름 대비 예시 1쌍을 추가한다 — "영입을 확정했습니다" (금지)
  → "영입을 확정했다" (올바름).
- 지시가 이미 있는데 무시되는 상황이므로, 규칙 서술을 늘리기보다 구체 대비 예시로 이탈을 줄인다.
- 3개 프롬프트 (`SUMMARY_PROMPT` · `TRANSLATE_PROMPT` · `PARAPHRASE_PROMPT`)에 동일 적용한다.

### 3. 백필 — 검출 기반 선별 재생성

- enrich 사이클에 말투 백필 단계를 추가한다: 검출기에 걸리는 기존 행을 회차당 상한 N건만 골라 요약을 재생성한다.
- 텍스트를 정규식으로 치환하지 않는다 — 어미 활용이 다양해 치환 규칙이 누더기가 되므로, 검출기는 "누구를 다시 쓸지"만 정하고 문장은 강화된 프롬프트를 든 LLM이 원문에서 다시 요약한다.
- 재생성 결과가 검출기를 통과하면 대상에서 자동으로 빠진다 — 멱등, 기존 하루 4회 스케줄에 편승.
- 생성 직후 재시도 로직은 두지 않는다 — 신규 생성분이 다시 합니다체로 나와도 다음 회차 백필이 잡는 구조로 단일화 (단순함 우선).
- 회차 상한 제안값은 20건 — 구현 시 검출 쿼리로 라이브 잔존 규모를 실측한 뒤 429 여유 내에서 조정한다.
- 재생성은 요약 필드만 갱신하고 `body_ko` 등 다른 enrich 결과는 건드리지 않는다.

## 에러 처리

- 429: 기존 규칙 그대로 — 식별 시 그 회차 즉시 중단 · WARNING 로깅 (파싱 실패와 구분), 남은 건은 다음 사이클 누적.
- 파싱 실패: 기존 enrich 경로의 처리를 재사용한다.

## 검증 · 완료 기준

- 단위: 검출기 케이스 3종 + 백필 선별 로직 (모킹).
- 라이브: 백필 수렴 후 검출 쿼리 0건 — 이 트랙의 종료 조건이자 v1 완성 선언 조건의 일부.

## 열어 둔 판단 (plan 단계 확정)

- `summary_ko`와 `summary3_ko` 재생성이 별도 호출인지 한 호출인지 — 현재 enrich 흐름 (ko/en 경로 분리)을 따라 확정한다.
- 검출 어미 목록의 최종 범위 — 단위 테스트 케이스로 고정한다.
