# UI 디자인 개편 설계 — 에디토리얼 문법 · 밀도 · 출처 강화 (2026-07-21)

공개 전환 이후 서빙 화면을 개편하는 설계.
현재 화면은 회색 배경에 흰 카드가 떠 있고 운영체제 기본 글꼴을 쓰는 조합이라, 내용과 무관하게 자동 생성물처럼 읽힌다.
여기에 한 화면에 들어오는 기사가 적고 기사 사이의 경중이 드러나지 않는다는 문제가 겹친다.
활자와 괘선으로 위계를 만들고 밀도를 높이며, 출처를 화면 앞으로 끌어내 신뢰를 확보하는 것이 이번 개편의 골자다.

## 1. 문제

- **시각 어휘가 기본값이다**
— 카드 + 그림자 + 12~14px 모서리 + 시스템 글꼴은 자동 생성 UI 가 가장 흔히 내놓는 조합이다.
- **색을 너무 많이 쓴다**
— 영입 단계 7개에 서로 다른 색 점을 붙여 한 화면에 일곱 색이 동시에 뜬다.
- **내부 용어가 독자에게 노출된다**
— 카드마다 붙는 `Tier 1.5` 는 파이프라인 내부 등급이지 독자용 표현이 아니다.
- **밀도가 낮다**
— 16:9 썸네일 + 요약을 담은 2열 카드라 첫 화면에 4~6건만 들어온다.
- **경중이 없다**
— 오피셜 확정 소식과 가십 라운드업이 같은 크기 · 같은 무게의 카드로 나열된다.
- **사이드바가 무겁다**
— 팀 · 영입 단계 · 소스 · 기자 · 신뢰도가 모두 펼쳐진 채 세로로 길게 늘어선다.
- **죽은 링크가 있다**
— 상단 내비의 '소개' · '일정' 이 `#` 로 연결돼 있다.

## 2. 목표 · 비목표

- 목표 A: 자동 생성물처럼 보이는 인상을 줄이고 스포츠 매체다운 고유한 인상을 만든다.
- 목표 B: 한 화면에 들어오는 기사 수를 늘리고 중요한 소식이 먼저 눈에 띄게 한다.
- 목표 C: 출처 · 기자 · 공신력 · 원문 링크를 화면 앞으로 끌어내 신뢰를 확보한다.
- 목표 D: 사이드바 필터를 정리하고 공신력을 축으로 삼아 탐색을 단순하게 만든다.
- 비목표: 운영 뷰 (`ops.html`) 디자인 통일 · 이벤트 로그 · 소스 확장 · 배포 · 인프라.
- 비목표: 수집 · 분류 계층 수정 (§16 에 별도 트랙 후보로 기록).

## 3. 방향 결정

목업 5안을 실데이터로 렌더해 비교한 뒤 **E안** 을 채택했다.

| 안 | 성격 | 결과 |
|---|---|---|
| A | 현행 톤 유지 + 톱스토리 + 3열 | 밋밋함이 그대로 남음 |
| B | 신문 마스트헤드 · 세리프 · 괘선 | 자동 생성 인상이 가장 옅음 · 밀도 낮음 |
| C | 다크 헤더 · 이적 보드 · 단계 트래커 | 관리자 대시보드처럼 읽힘 |
| D | A 기반 + 카드 한 줄 요약 + 2열 가로형 | 가독성 · 밀도 균형이 가장 좋음 · 인상은 A 와 동일 |
| E | **D 의 구조 + B 의 활자 · 괘선 문법** | **채택** |

채택 근거는 구조와 문법을 분리해 각각의 장점을 취할 수 있다는 것이다.
D 가 만든 레이아웃 (톱스토리 밴드 + 2열 가로형 목록 + 접히는 사이드바) 은 그대로 두고, 그 위에 B 의 시각 문법 (세리프 제목 · 괘선 · 색 절제) 을 얹었다.

## 4. 디자인 토큰

### 4.1. 색

강조색은 셋으로 제한한다.
빨강은 확정 · 초록은 진행 · 노랑은 선수 측 합의를 뜻하고, 나머지는 모두 무채색이다.

| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--red` | `#DA020E` | `#FF4F57` | 오피셜 · 이적 합의 · 링크 · 강조 |
| `--green` | `#1D7A42` | `#54C97C` | 협상 중 |
| `--yellow` | `#A8710A` | `#E0A83C` | 개인 합의 |
| `--ink` | `#16130f` | `#efeae2` | 본문 |
| `--mut` | `#6f6960` | `#a49d94` | 보조 텍스트 |
| `--dim` | `#928c82` | `#7d766d` | 메타 · 캡션 |
| `--paper` | `#fbfaf7` | `#131211` | 배경 |
| `--sunk` | `#f2efe9` | `#1d1b19` | 입력창 · 이미지 자리 · 원문 박스 |
| `--hair` | `#ddd8ce` | `#332f2b` | 얇은 구분선 |
| `--rule` | `#16130f` | `#efeae2` | 굵은 구분선 |

### 4.2. 활자

- 제목 · 워드마크 · 인용은 세리프 (`Noto Serif KR` 700 · 900).
- 본문 · UI 는 산세리프 (`Pretendard`).
- 한국어 장문은 화면에서 세리프 가독성이 떨어지므로 상세 본문은 산세리프를 쓴다.
- **글꼴은 자체 호스팅한다**
— 목업은 Google Fonts · jsDelivr CDN 을 썼으나, 정적 배포 (Cloudflare Pages) 환경에서 외부 CDN 두 곳에 의존하면 렌더가 남의 가용성에 묶인다.
한국어 서브셋 `woff2` 를 `serve/static/` 에 두고 `@font-face` 로 불러온다.
- 글꼴 로딩 실패에 대비해 `font-family` 폴백에 시스템 글꼴을 남긴다.

### 4.3. 형태 · 다크 모드

- 떠 있는 카드와 그림자를 쓰지 않는다.
구역은 괘선과 여백으로 나눈다.
- 모서리 반경은 0~2px 로 제한한다.
- 이모지를 쓰지 않는다.
아이콘은 인라인 SVG 로 그린다.
- 다크 모드는 `prefers-color-scheme` 을 기본으로 삼고 `html[data-theme]` 로 덮어쓴다.
선택은 `localStorage` 에 저장하고, 첫 페인트 전 인라인 스크립트로 적용해 깜빡임을 막는다.

## 5. 영입 단계 표시 규칙

**저장 enum 은 건드리지 않는다.**
`transfer_stage.py` 는 enrich 의 분류 프롬프트 · 검증과 공유하는 단일 출처라, 이 트랙은 표시 계층에서만 묶고 정렬한다.

| 표시 순서 | 라벨 | 포함 enum | 색 | 형태 |
|---|---|---|---|---|
| 1 | 오피셜 | `official` | 빨강 | 채움 배지 |
| 2 | 이적 합의 | `agreed` | 빨강 | 글자 |
| 3 | 협상 중 | `negotiating` · `medical` | 초록 | 글자 |
| 4 | 개인 합의 | `personal_terms` | 노랑 | 글자 |
| 5 | 관심 | `interest` | 회색 | 글자 |
| 6 | 루머 | `rumour` | 회색 | 글자 |

- 순서는 진행이 많이 된 것부터다.
선수 측 개인 합의가 구단 간 협상보다 먼저 이뤄지는 사례가 흔해 협상 중을 개인 합의 위에 둔다.
- 메디컬은 건수가 적고 (전체 3건) 독자에게 따로 나눌 실익이 낮아 협상 중에 합친다.
- 사이드바에는 색 점을 붙이지 않는다.
라벨만 두고 색은 기사 배지에서만 쓴다.

## 6. 인덱스

### 6.1. 톱스토리 밴드

- 좌우 2열 (`1.62fr` · `1fr`) 로 리드 1건과 주요 소식 4건을 배치한다.
- 리드 선정은 영입 단계를 우선한다
— `official` → `agreed` → `medical` → `personal_terms` → `negotiating` 순으로 점수를 매기고, 같은 단계 안에서는 최신순으로 고른다.
여기 쓰는 값은 저장 enum 이다 (§5 의 화면 묶기와 무관하게 메디컬을 독립 순위로 둔다).
- 최근 10일 이내 기사만 후보로 삼는다.
- 리드 이미지는 21:9 로 자른다.
첫 화면에 최신 소식 첫 줄이 걸리도록 높이를 제한한 값이다.
- 리드에는 단계 · 제목 (세리프 28px) · 리드문 · 언론사 · 공신력 · 시각을 싣는다.
- 주요 소식 4건은 88px 썸네일 (3:2) 과 두 줄 제목으로 괘선 구분 목록을 만든다.

### 6.2. 최신 소식

- 2열 가로형 목록이다.
왼쪽에 132px 썸네일 (3:2), 오른쪽에 제목 두 줄 · 요약 두 줄 · 메타 한 줄을 둔다.
- 카드 테두리를 쓰지 않고 항목 사이를 얇은 괘선으로만 나눈다.
- **날짜로 묶는다**
— 오늘 · 어제 · `7월 19일 (일)` 순으로 구분선을 넣고 그날의 건수를 함께 적는다.
- 섹션 제목 오른쪽 끝에 `KST 기준` 을 작게 표시한다.

## 7. 사이드바 필터

- 순서는 팀 → 영입 단계 → 공신력 → 소스 (언론사) → 기자 로 바꾼다.
- 영입 단계와 공신력은 기본으로 펼치고, 소스와 기자는 기본으로 접는다.
- 상단에 있던 단계 탭 바는 사이드바와 중복이라 없앤다.
헤더가 두 줄로 줄어 첫 화면에 기사가 더 들어오는 이득도 있다.
- 스크롤바가 오른쪽 건수를 가리지 않도록 `scrollbar-gutter: stable` 로 자리를 미리 비운다.
- 상태줄과 버튼 줄의 위 여백을 다른 그룹과 같게 맞춘다.

### 7.1. 공신력 — 내부 등급을 독자 표현으로

| 저장 tier | 독자 표기 |
|---|---|
| 0 | 구단 공식 |
| 1 | 공신력 최상 |
| 1.5 | 공신력 상 |
| 2 | 공신력 중 |
| 3 | 공신력 하 |
| 4 | 공신력 최하 |

- 카드 · 상세 · 사이드바 어디에서도 `Tier 1.5` 같은 문자열을 노출하지 않는다.
- 맨 위에 '전체' 를 두고 기본 선택으로 삼는다.
- 등급은 체크박스로 여러 개를 동시에 고를 수 있다.
하나라도 고르면 '전체' 가 풀리고, 모두 풀면 '전체' 로 돌아간다.

### 7.2. 공신력 ↔ 소스 · 기자 연동

- 공신력을 고르면 그 등급에 속한 소스와 기자가 자동으로 체크되고, 접혀 있던 두 그룹이 함께 펼쳐진다.
'더보기' 안에 숨어 있던 항목이 체크 대상이면 그 구간도 자동으로 노출한다.
- 예를 들어 공신력 최상을 고르면 소스에서 The Athletic, 기자에서 David Ornstein · Sami Mokbel 이 체크된다.
- 자동 체크된 항목은 개별로 해제할 수 있다.
하나라도 손대면 상태줄에 `직접 고름` 을 덧붙여 사용자가 조정했음을 알린다.
- 초기화 버튼은 공신력 · 소스 · 기자를 모두 되돌린다.
- 자동 체크는 결과를 바꾸는 장치가 아니라 **그 등급에 무엇이 들어가는지 보여 주고 거기서 좁혀 가게 하는 출발점** 이다.
- 이 연동이 성립하려면 소스 · 기자 패싯의 각 항목이 자기 등급을 들고 있어야 한다.
템플릿이 항목마다 `data-tier` 를 함께 내보내고, 등급 미상 항목은 빈 값으로 둔다.

### 7.3. 등급이 걸치는 문제 — 이미 해소돼 있다

`config/credibility.yaml` 이 규칙을 주석으로 명시하고 있다.
언론사 등급은 그 매체의 비전담 기사에 적용되는 기준선이고, 등재된 전담 기자가 쓴 기사는 기자 등급으로 승격된다 (BBC 는 1.5 이지만 Sami Mokbel 은 1).
따라서 기사 하나의 등급은 "등재 기자가 있으면 기자 등급, 없으면 언론사 등급" 으로 항상 하나로 정해진다.
공신력 최상을 골랐을 때 BBC 가 체크되지 않는 것이 오히려 정확하다.
BBC 전체가 최상은 아니기 때문이다.

## 8. 필터 결합 규칙 변경 — 소스와 기자를 OR 로

현재 `app.js` 는 모든 필터 그룹을 AND 로 묶는다.

```js
const visible = okText && okOutlet && okTier && okJournalist && okStage;
```

이 상태로 §7.2 의 자동 체크가 들어가면 결과가 어긋난다.
공신력 최상을 고르면 소스는 `{The Athletic}`, 기자는 `{Ornstein, Mokbel}` 이 되는데, AND 로 묶으면 Mokbel 이 BBC 에 쓴 기사가 소스 조건에서 탈락해 사라진다.
The Athletic 의 미등재 기자 기사도 마찬가지로 탈락한다.

**소스와 기자를 OR 로 묶고, 그 결과를 나머지 축과 AND 로 결합한다.**

```
(소스 조건 OR 기자 조건) AND 공신력 AND 영입 단계 AND 검색어
```

- 두 축 모두 "이 기사가 어디서 왔는가" 를 가리키므로 OR 이 의미상으로도 맞다.
- 이렇게 하면 Mokbel 의 BBC 기사가 정상적으로 걸리고, 사용자가 Mokbel 을 해제하면 그 기사만 빠진다.
- 소스와 기자 중 어느 쪽도 선택되지 않았을 때는 이 항이 조건에서 빠진다 (기존 동작 유지).
- URL 계약 (`?outlet=..&journalist=..&tier=..&stage=..`) 은 그대로 둔다.
바뀌는 것은 읽어 들인 값을 결합하는 방식뿐이다.
- 상태줄의 조건 개수 표기도 바뀐 결합에 맞춰 갱신한다.

## 9. 상세 페이지

**본문 전문 번역 서빙을 전제로 설계한다.**
언론사 6종을 발췌에서 전문으로 되돌리는 작업은 별도 트랙이 진행 중이다.

배치 순서는 다음과 같다.

1. 목록으로 돌아가는 링크
2. 단계 배지
3. 제목 (세리프 36px)
4. **원문 제목 (영문 원제)** — 회색 작은 글씨로 병기
5. 리드문 (한 줄 요약)
6. 메타 그리드 — 언론사 · 기자 · 공신력 · 발행일을 괘선 사이에 라벨과 함께
7. **자동 번역 고지** — "이 글은 원문을 자동 번역한 것입니다. 표현이 어색하거나 뜻이 모호하면 원문을 확인해 주세요."
8. 히어로 이미지
9. 핵심 요약 — 붉은 상단 괘선 · 제목 옆에 `자동 생성` 표시
10. 본문 — 17px · 행간 1.88
— BBC 가십 라운드업은 문단이 단신 여러 건을 이어 붙인 형태라, 단신 단위로 끊고 각 단신 끝의 출처를 분리해 표시한다 (§9.1).
11. **원문 박스** — 언론사 · 기자 · 날짜와 함께 원문 링크를 버튼으로 강조
12. 함께 볼 소식
13. 저작권 · 자동 번역 안내 푸터

4 · 7 · 9 · 11 이 신뢰 확보의 핵심이다.
이 서비스의 본문은 실제로 기계 번역이므로, 숨기면 독자가 어색한 문장을 만났을 때 질 낮은 자동 생성물로 결론짓는다.
먼저 밝히면 같은 문장이 번역본으로 읽힌다.

### 9.1. 라운드업 단신 표시 — PR #105 를 새 문법으로 흡수한다

PR #105 (`feat/gossip-item-badges`) 가 이미 넣은 기능이다.
판정 로직은 그대로 두고 시각 표현만 이 트랙의 문법으로 다시 그린다.

- **판정 로직은 건드리지 않는다** (§15)
— 문단 끝 괄호를 그대로 믿지 않고 원문의 `(출처) , external` 표지 집합에 앵커해 변환하므로, 라운드업 뒤쪽 일정 섹션의 경기장 · 시각 괄호는 변환되지 않는다.
디자인을 바꿔도 그대로 필요한 자산이다.
- **시각 표현은 다시 그린다**
— #105 의 `.gossip-item` 은 모서리 반경 9px 의 테두리 상자이고 `.src-tag` 는 반경 99px 의 알약 배지다.
§4.3 의 "떠 있는 카드 없음 · 반경 0~2px" 와 정면으로 어긋난다.
- **옛 토큰을 옮겨 쓸 수 없다**
— #105 가 쓰는 `--navy` · `--chip` 은 §4.1 새 토큰 표에 없다.
블록만 그대로 옮기면 정의되지 않은 변수를 참조하게 되므로 색을 새로 정해야 한다.
- 새 표현
— 단신은 테두리 상자 대신 좌측 2px 괘선과 들여쓰기로 끊고, 출처는 알약 배지 대신 사이드바 등급 라벨과 같은 계열의 작은 회색 글자로 둔다.
- 템플릿의 `{% elif b.type == 'item' %}` 분기는 유지한다.
없애면 라운드업 본문이 통짜 문단으로 돌아간다.

## 10. 소개 페이지 · 내비게이션

- 상단 내비에서 '일정' 을 없앤다.
경기 일정 데이터 소스가 파이프라인에 없어 지금은 채울 수 없고, 소스 확장은 이번 트랙의 비목표다.
- '소개' 는 실제 페이지 (`about.html`) 로 만든다.
- 담을 내용은 서비스 목적 · 수집하는 소스의 구성 · 공신력 등급 체계 설명 · 자동 번역 고지 · 저작권 정책이다.
- 인덱스와 상세 하단에도 저작권과 자동 번역을 한 문단으로 요약해 둔다.

## 11. 이미지 · 썸네일

- 썸네일은 배경 이미지 대신 `<img>` 와 `object-fit: cover` 로 그린다.
바깥 상자에 비율을 고정해 이미지 크기를 통일한다.
- **비율이 무시되던 문제를 고친다**
— 가로형 행이 flex 컨테이너인데 기본값이 `align-items: stretch` 라 이미지 높이가 옆 텍스트 길이에 맞춰 늘어났다.
`align-self: flex-start` 로 고정한다.
- 이미지가 없거나 로드에 실패하면 같은 비율의 자리를 유지한 채 옅은 `Bullet-in` 표시를 넣는다.
실패는 `onerror` 로 감지해 `<img>` 를 제거하고, CSS 는 `:empty` 와 `.noimg` 두 경우에 플레이스홀더를 그린다.
- 문서에 `<meta name="referrer" content="no-referrer">` 를 두고 각 이미지에도 `referrerpolicy="no-referrer"` 를 붙인다.

## 12. 시각 · 날짜 표시

- 저장은 UTC 이므로 표시 단계에서 한국 시간으로 바꾼다.
날짜 그룹의 경계와 상단 날짜도 같은 기준으로 맞춘다.
- **날짜 그룹 안에서만 시각을 쓴다**
— 최신 소식 목록은 이미 날짜로 묶여 있으므로 항목마다 `22:37` 처럼 시각만 보여 준다.
- 톱스토리 밴드의 리드와 주요 소식은 날짜 그룹 밖에 있으므로 기존 상대 표기 (`1일 전`) 를 그대로 쓴다.
- **발행 시각 정밀도 결정을 그대로 따른다**
— `published_precision` 이 `day` 인 기사는 시각을 지어내지 않고 비워 둔다.
`time` 이거나 값이 없으면 시각을 보여 준다 (`2026-07-20-published-at-accuracy-design.md` §4.1 · §4.3).
- 추출에 실패해 수집 시각으로 폴백한 소스 (기사 페이지를 가져오지 않는 BBC 가십 등) 는 수집 시각이 그대로 표시된다.
이는 해당 스펙 §2 가 비목표로 명시해 받아들인 동작이다.

## 13. 모바일

- 1000px 이하에서 톱스토리 밴드가 1열로 쌓이고, 최신 소식도 1열이 된다.
사이드바는 숨기고 햄버거 버튼으로 연다.
- 640px 이하에서 검색창과 정렬 선택을 숨기고 제목 크기를 줄인다.
- 가로형 목록 배치는 모바일에서도 유지한다.
세로 카드보다 한 화면에 더 많이 들어온다.

## 15. 소유권 경계 — 건드리지 않는 것

병렬 세션과 충돌하지 않도록 다음은 수정하지 않는다.

- `config/sources.yaml` 의 `serving` 값.
- `render.py` 의 `serving_mode()` · `_excerpt` 관련 줄.
- `detail.html.j2` 의 `excerpt-note` 블록 — 자리만 유지하고 리디자인하지 않는다.
- `quality.py` · `enrich.py` · `transfer_stage.py`
— `enrich.py` 의 `attrib_core` · `roundup_attrib_counts` 는 아래 `gossip_itemize()` 가 의존한다.
- `render.py` 의 `gossip_itemize()` 와 `render_article()` 안의 `bbc_gossip` 분기 (PR #105)
— 출처 표지 앵커 판정은 시각과 무관한 자산이라 개편 대상이 아니다.
표시 방식만 §9.1 에서 다시 그린다.
- `infra/` · `README.md` · `docs/runbook` 의 vm-cohost · afcstuff 런북.

## 16. 범위 밖 — 별도 트랙 후보

이번 개편 중 발견했으나 표시 계층에서 고칠 수 없는 것들이다.
셋 다 독자 눈에는 "아무거나 긁어오는 사이트" 로 보이게 만드는 문제라 우선순위가 낮지 않다.

### 16.1. 아스날 공식 홈페이지 이미지가 브라우저에서 차단된다

- `assets.arsenal.com` 이미지는 curl 로는 100KB 가 정상 수신되지만 크롬은 `ERR_BLOCKED_BY_ORB` 로 거부한다.
- 번들 Chromium 과 실제 Chrome 양쪽, 그리고 정상 출처 (localhost) 에서도 같은 결과다.
라이브 사이트에서도 같은 자리가 비어 있다.
- 이번 트랙은 §11 의 플레이스홀더로 자리만 지킨다.
- 근본 해결은 이미지를 자체 캐시해 서빙하는 것인데, 저장소와 배포 비용이 따르는 인프라 결정이다.

### 16.2. 출처가 `NA` 로 표시된다

- fmkorea 말머리 `[NA]` 를 언론사명으로 그대로 저장한 결과다.
실제 출처는 NowArsenal 이다.
- `src/bullet_in/adapters/fmkorea.py` 의 `OUTLET_MAP` 에 `"NA": "NowArsenal"` 한 줄을 더하면 이후 수집분은 해결된다.
- 이미 저장된 행은 바뀌지 않으므로 백필 여부를 따로 판단해야 한다.
- 수집 계층 수정이라 이 트랙의 소유 범위 밖이다.

### 16.3. 아스날과 무관한 기사가 노출된다

- fmkorea 게시판에는 타 구단 소식도 올라오는데, 관련성 판별이 없어 `team=arsenal` 로 저장되고 그대로 실린다.
- 실제 사례로 노팅엄 포레스트의 크사버 슐라거 영입 기사가 루머로 분류돼 목록에 떠 있다.
- 수집 · 분류 계층 문제라 표시 계층에서는 손댈 수 없다.
