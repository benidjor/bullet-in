# fmkorea 발견 소스 — 원문 URL 추출 함정 & 게시판 공지 의존

- **날짜**: 2026-06-29
- **영역**: adapters (fmkorea)
- **심각도**: 중 (원문 오인 시 엉뚱한 기사 번역 · dedup 병합 실패)

## 증상

fmkorea 를 "원문 발견 소스"로 전환 (Tier 2-a)하며 글의 원문 URL 을 추출할 때, 일부 글에서 기사가 아니라 기자 프로필 / 임베드 링크가 원문으로 잡힘.

- 실측 post `10007542458` — 본문에 기자 프로필 앵커 (`nytimes.com/athletic/author/david-ornstein/`)와 끝에 실제 기사 평문 URL 이 함께 존재.
- 기존 추출기가 **프로필 앵커**를 반환 (기사 URL 아님).

## 진단 과정

fmkorea 글을 단독 fetch 해 `_extract_original_url` 동작을 격리 확인.

```
기존 우선순위
  el.select("a[href]") 앵커
        │  (광고 · 기자 프로필 · 트위터 임베드가 본문 중간에 먼저 등장)
        ▼
  잘못된 첫 앵커를 원문으로 반환

fmkorea 관례 (게시판 공지가 강제)
  출처 링크 = 본문 끝 평문 텍스트 (앵커 아님)
```

## 원인

두 축의 어긋남.

- 추출 우선순위 (앵커 우선)가 fmkorea 관례 (끝쪽 평문)와 반대.
- 발견 흐름 전체가 **게시판 공지 규칙**에 의존
  - 말머리 `[언론사]` 필수 · 출처 링크 위치 · 인용 / SNS 금지 · 유료 매체 셋
  - 공지는 2018 → 2025 로 개정되어 옴 (구조 고정이 아님)

## 해결

- `_extract_original_url` 을 **끝쪽 평문 URL 우선**으로 수정 (평문 여럿이면 마지막), 앵커는 폴백 (`aa5b19e`).
- 말머리 파싱 (`parse_bracket`) · `PAYWALLED_OUTLETS = {"The Athletic"}` · 인용 / SNS 금지 전제는 모두 공지 기반임을 코드 주석 · spec 에 명시.

## 예방 / 재점검 신호

fmkorea 결과 품질이 떨어지면 (원문 오인 · 스킵 급증) 게시판 공지부터 재확인.

- 말머리 `[언론사]` 형식 변화
- 출처 링크 위치 변화 (끝 평문에서 앵커 / 단축 URL 로)
- 인용 / SNS 허용 정책 변화 (원문이 트윗으로 바뀜)
- 유료 매체 셋 변화 (디 애슬레틱 외 추가)

보조 원칙.

- 셀렉터 드리프트 (`docs/troubleshooting/2026-06-12-live-source-selector-drift.md`)의 소식통 버전 — 단위 테스트는 모킹이라 못 잡으므로 실글 단독 fetch 로 검증.
- 참조 공지: `818271708` · `8677095961` · `9299550074`.
