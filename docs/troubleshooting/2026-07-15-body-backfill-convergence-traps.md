# 본문 말투 백필 수렴 정체 — 잔존을 원인별로 분리해야 끝난다

본문 표시 컨벤션 트랙 (#52) 에서 `body_ko` 존댓말 59건을 재번역 백필로 수렴시키는 중,
3차 패스에서 잔존이 13건에서 멈췄다.
같은 프롬프트로 재번역을 반복해도 줄지 않는 구간이 생기면, 잔존 표본을 열어 원인을 분리해야 한다.

## 증상

- 패스별 잔존: 59 → 23 → 15 → **13 (정체)**.
- 재번역 자체는 성공하는데 검출기에 다시 걸림 — LLM 지시 무시로 보이지만 전부는 아니었다.

## 원인 — 잔존 13건은 3종 혼합이었다

1. **검출기 오탐 — 평어체 '아니다'**:
   `~니다` 패턴이 '쉬운 과제가 **아니다**' 같은 평어체 문장을 잡았다.
   재번역해도 같은 문장이 나오므로 영원히 수렴하지 않는 클래스.
2. **따옴표 없는 콜론 인용**:
   LLM이 발화를 "홉킨슨은 이렇게 말했다: …할 것입니다" 형태 (따옴표 없음) 로 번역
   → 인용부호 예외가 안 걸려 정상 발화 존댓말이 검출됨.
3. **홍보 문구 잔존**:
   "구독 · 앱 설치" 예시만으로는 SNS 팔로우 요청 · 팟캐스트 · 뉴스레터 홍보를 LLM이 못 걸렀다.

## 해결 — 각 원인을 다른 계층에서 교정

- ① 검출기: `(?<!아)니다` — '아니다' 제외, '아닙니다'는 '닙니다'로 끝나 계속 검출 (회귀 테스트 2종).
- ② 프롬프트: "발화 인용은 반드시 큰따옴표로 감싼다" — 재번역 시 인용 예외가 걸리는 형태로 유도.
- ③ 프롬프트: 무관 문구 예시에 SNS 팔로우 · 팟캐스트 · 뉴스레터를 명시 열거.
- 교정 후 4~6차 패스로 **잔존 0/166** 수렴 (Gemini 503 스킵분은 재시도 패스가 흡수).

## 같은 세션의 인접 발견 2건

- **한글 · 라틴 혼용 전사** ('마ikel Arteta'): LLM이 이름 전사 도중 스크립트를 섞는 저빈도 결함.
  검출 쿼리 — 한글 음절 바로 뒤 소문자 라틴은 정상 표기 (FA컵 등 대문자 약어) 와 달리 의심 신호:
  ```python
  re.compile(r"[가-힣][a-z][A-Za-z]*")   # 전수 스캔에서 1건만 검출·수정
  ```
- **비통용 선수 표기** (모르간 로저스 · 스캇 · 메슬리에):
  재발 방지는 `config/glossary.yaml` (오표기 → 통용) + `enrich.apply_glossary` 후처리 치환.
  말투와 달리 고유명사 치환은 결정적이라 LLM 재생성이 필요 없다.

## 교훈 · 예방

- 백필 수렴이 정체되면 패스를 더 돌리기 전에 **잔존 표본을 직접 읽어 원인을 분류**한다
  — 검출기 오탐 · 표기 형태 문제 · 진짜 지시 무시는 처방이 다르다.
- 검출기 어미 패턴에 새 어미를 넣을 때는 **평어체와 겹치는 접미 (아니다 류) 를 함께 점검**한다.
- LLM에게 "제외하라"는 지시는 범주 서술보다 **구체 예시 열거**가 잘 듣는다 (트랙 ① 대비 예시와 동일 교훈).
