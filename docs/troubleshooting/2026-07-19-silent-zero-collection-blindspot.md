# 평시 0건 소스의 조용한 수집 드리프트 — 감시 2종이 모두 못 잡는 사각

- 날짜: 2026-07-19 (트랙 ⑤ 진행 중 발견)
- 증상 소스: arsenal_official (공홈)
- 관련: 백로그 SoT `docs/superpowers/2026-07-19-post-v1-followup-tracks.md` §6 (복구 항목)

## 1. 증상

- 공홈 방출 오피셜 기사 (트로사르 → 베식타스) 가 화면에 없다는 사용자 질문으로 발견됐다.
- 어댑터 단독 프로브 결과 arsenal_official 은 **셀렉터 매칭 0건** — 사이트 전면 개편 시점부터 조용히 0건 수집 중이었다.
- 그동안 어떤 알림도 발생하지 않았다.

## 2. 왜 감시 2종이 모두 놓쳤나

수집 감시는 두 축인데, **"평시에도 0건인 소스"** 는 둘 다의 구조적 사각이다.

- **SLO-6 수집량 이상 (volume_anomalies)** — 소스별 최근 이력 대비 2σ 이탈을 본다.
  arsenal_official 은 'sign' 고정밀 필터로 평시 대부분 회차가 0건
→ 기준선 자체가 0 이라 "계속 0건" 은 이탈이 아니다.
- **SLO-5 신선도 (evaluate_freshness)** — 소스별 `MAX(fetched_at)` 워터마크 경과를 본다.
  기사 적재가 0건인 소스는 워터마크가 NULL 이고, NULL 은 신규 소스 오탐 방지를 위해 **행 기록 · 알림 제외** 로 설계됐다
→ 한 번도 적재된 적 없는 소스는 영원히 알림 대상 밖이다.
- 요약: 두 감시 모두 "정상적으로 볼륨이 낮은 소스" 와 "죽은 소스" 를 구분하지 못한다.
  저볼륨 · 고정밀 필터 소스일수록 드리프트가 오래 숨는다.

## 3. 진단 절차 (실측)

1. 어댑터 단독 fetch 프로브 — 목록 셀렉터 매칭 수를 확인한다 (0건이면 드리프트 의심).
2. 목록 페이지를 직접 fetch 해 DOM 을 본다 — 구 셀렉터 존재 여부 · class 체계 변화 · 본문 영역 링크 유무.
3. 실측 (2026-07-19): 구 셀렉터 `a.responsive-card__wrapper` 0건, 신규 사이트는 `MegaNav_*` 클래스 체계 · **목록이 클라이언트 렌더링**
→ 147KB HTML 에 기사 링크가 0건이라 정적 fetch 로는 복구 불가 (Playwright 또는 데이터 API 필요, goal 복구 선례 규모).

## 4. 대응 · 예방

- 복구는 백로그 SoT §6 항목으로 처리한다 ('sign' 필터의 방출 차단 재설계 포함).
- 저볼륨 소스는 정기 점검에 **어댑터 단독 프로브** 를 포함한다 — 자동 감시가 못 잡는다는 사실을 전제로 한 수동 안전망.
- 감시 보강 아이디어 (백로그 제안): fetch 는 성공했는데 셀렉터 후보가 0건인 회차를 별도 이벤트로 로깅하면 이 사각을 자동화로 좁힐 수 있다.
  단 도입 여부는 별도 판단 — 지금은 기록만 한다.
