# 구현 subagent 의 파일 전체 바이트 치환 — 태스크 밖 기존 코드 오염 함정

- 날짜: 2026-07-20 (구단명 검출기 트랙 Task 1, 컨트롤러 검수에서 발견 · 커밋 폐기로 해소)
- 결함 클래스: subagent 전사 실패 (`2026-07-15-subagent-unicode-transcription-loss.md` 와 같은 계열의 다른 모드)
- 해소: 오염 커밋 폐기 (reset) 후 제약 강화 재디스패치 — 최종 브랜치 (PR #91) 에는 오염 흔적 없음

## 1. 증상

- 구현 subagent (Haiku) 가 DONE 으로 보고한 커밋의 diff 에 **태스크 범위 밖 기존 줄 다수가 변경**돼 있었다.
  - 기존 docstring · 주석의 `—` 가 전부 `-` 로 치환.
  - **`TRANSLATE_PROMPT` (Gemini 에 실제로 나가는 프롬프트 문자열) 내용까지 변경**.
  - `_SENT_END_RE` 정규식의 닫는 곱슬따옴표 `”` 가 ASCII `"` 로 치환
  → 곱슬따옴표로 끝나는 문장의 경계 판정이 실제로 바뀌는 회귀.
- 신규 코드의 docstring 도 계획 원문에서 예시가 탈락하는 등 손실 전사.

## 2. 원인 — 자기 오염 후 광역 "수정"

- 계획 파일의 코드 블록은 순수 ASCII 따옴표였다 (실측 곱슬따옴표 0개)
— 즉 원본에는 문제가 없었다.
- subagent 가 전사 과정에서 **스스로 곱슬따옴표를 타이핑해 넣고**, 그로 인한 오류를 인지하자
  bash 스크립트로 **파일 전체에 UTF-8 바이트 치환**을 돌려 "수정" 했다.
- 자기가 만든 문제의 수정 범위를 자기 변경분으로 한정하지 않아, 기존 코드의 정당한 특수문자까지 쓸었다.

## 3. 왜 테스트가 못 잡았나

- 전체 테스트는 통과 상태였다 (55 passed)
— 문장 경계 테스트의 픽스처가 ASCII 따옴표만 사용해 정규식 회귀가 드러나지 않았고,
  프롬프트 문자열은 내용 변경을 검증하는 테스트가 없다 (LLM 산출 품질에만 반영되는 값).
- 즉 "테스트 통과" 는 이 오염 클래스에 대한 안전 신호가 아니다.

## 4. 발견 경위 — 보고 문구가 사고 신호였다

- DONE 보고의 두 문구가 레드 플래그였다:
  "Initial markdown curly quotes required UTF-8 byte replacement" · "Docstring simplified for compatibility".
- 컨트롤러가 이 문구를 신호로 diff 를 실측 (`git diff base..head`) 해 기존 줄 변경을 확인했다.
- 교훈: **"단순화했다 · 정리했다 · 치환으로 해결했다" 류의 자기 보고는 검수 트리거**다
— 성공 보고처럼 읽히지만 실제로는 일탈의 자백일 수 있다.

## 5. 해결

1. 오염 커밋 폐기 — `git reset --hard <base>` (베이스가 클린해 전체 재시도가 부분 수정보다 안전).
2. 재디스패치 (Sonnet) 에 제약 3종을 명시:
   - bash · sed 류 치환 스크립트 금지 — 파일 수정은 Edit 도구 (정확한 old/new 문자열) 만.
   - **diff = 삽입만** — 커밋 전 `git diff` 로 기존 줄 변경 0 을 자기 검증 (`--stat` 삽입만 + 삭제 줄 grep 0).
   - 커밋 전 위치 검증 (브랜치 · HEAD 확인) — 기존 가드.
3. 재시도 결과: 삽입 97줄 · 삭제 0줄, 원문 문자 단위 일치 확인 후 진행.

## 6. 예방 규칙

- **전사 태스크 디스패치에 위 제약 3종을 표준으로 포함**한다 — 사고 후가 아니라 처음부터.
- 컨트롤러는 DONE 보고를 그대로 믿지 않고, **삽입-온리 태스크는 삭제 줄 0 을 실측**한다
  (`git diff base..head | grep '^-' | grep -v '^---' | wc -l`).
- **한국어 · 특수문자 비중이 큰 전사 태스크는 Haiku 배정을 피한다** — 모델 배분 규칙의 예외로 등재.
- 계획 파일 작성 시 코드 블록을 ASCII 따옴표로 유지한다 (이번 계획은 준수 상태였음 — 원인 차단이 아니라 진단 단순화 목적).
