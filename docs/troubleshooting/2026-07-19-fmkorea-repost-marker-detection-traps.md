# fmkorea 퍼가기 표식 감지 확정 함정 — 가짜 음성 대조 · 본문 셀렉터 밖 표식 (2026-07-19)

퍼가기 금지 정책 (PR #85) 의 표식 감지기 `_is_repost_blocked` 를 실DOM 로 확정하는 과정에서 밟은 함정 2개의 기록.
표식 하나를 감지하는 단순 작업처럼 보이지만, 잘못 확정하면 오탐 (정상 글 강등) 또는 미탐 (금지 글 본문 복제 재개) 이 조용히 발생한다.

## 1. 함정 ① — 음성 대조 표본이 음성이 아님 (동일 작성자 편향)

spec 예시 글 (9940576222) 에서 표식을 찾은 뒤, 그 페이지 사이드바의 이웃 글 하나를 "일반 글" 로 골라 음성 대조를 시도했다.

- 결과: 그 글에도 표식 존재 — 같은 작성자 (디 마르지오 번역러) 의 금지 글이었다.
- 이 시점에서 표본이 양성 2 · 음성 0
→ **"표식이 전 페이지 템플릿에 상시 렌더되고 표시 여부만 CSS 토글" 가설을 배제할 수 없다.**
  상시 렌더라면 문자열 · 구조 검사 모두 전 글을 금지 글로 오탐한다.
- 해소: 성격이 다른 글 (다른 게시판 · 구 srl 안내글) 로 진짜 음성을 확보
→ 표식 부재 확인으로 조건부 (기능 생성) 렌더링 확정.

**교훈: 소식통 게시판 이웃 글은 작성자 · 유형이 편향돼 음성 표본이 못 될 수 있다.**
음성 대조는 "표식이 없을 것으로 기대되는 글" 이 아니라 **성격 자체가 다른 글**로 해야 배제력이 생긴다.
표식류 감지기는 양성 N건만으로 확정하지 말 것 — 음성 1건이 상시 템플릿 가설을 죽이는 유일한 증거다.

## 2. 함정 ② — 표식이 본문 셀렉터 밖이라 본문 검사로는 감지 불가

표식 `<strong>[퍼가기가 금지된 글입니다 - …]</strong>` 의 실측 위치는 `.rd_body` 직하위 · 본문 (`.xe_content`) **밖**이다.

- 어댑터의 본문 추출 (`_body_text(html, ".xe_content")`) 결과에는 표식 텍스트가 **포함되지 않는다**
→ "수집된 본문에서 문구를 찾는다" 식 감지는 항상 미탐.
- 반대로 전체 HTML 문자열 검색은 본문 · 댓글이 표식 문구를 **인용**하는 글에서 오탐.
- 해결: 구조 스코프 감지 — `.rd_body` 하위 `strong` 중 `.xe_content` 내부가 아닌 것에서만 문구를 찾는다 (댓글 영역은 `.rd_body` 밖이라 자연 배제).
  본문 인용 오탐은 회귀 테스트 (`test_is_repost_blocked_false_when_phrase_quoted_inside_body`) 로 고정.

**교훈: 페이지 레벨 표식은 "본문 어딘가에 있겠지" 가 아니라 실DOM 로 조상 체인을 확인해 스코프를 정한다.**
같은 함정의 변형 — 표식 위치 · 문구가 바뀌면 감지가 조용히 꺼진다 (드리프트 감시 신호는 운영 런북 참조).
